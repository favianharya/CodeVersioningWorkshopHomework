merging

Merge pada GitHub adalah proses menggabungkan perubahan yang ada di satu cabang (branch) ke cabang lain, 
seperti menggabungkan perubahan dari cabang fitur ke cabang utama (biasanya disebut main atau master). 
GitHub menyediakan antarmuka web yang memungkinkan pengembang untuk melakukan merge dengan mudah tanpa 
perlu menggunakan terminal atau Git secara langsung.

Fungsi Utama: Menggabungkan perubahan dari satu cabang ke cabang lain dalam repositori Git.

Manfaat:
- Mencegah duplikasi kode dengan menggabungkan perubahan dari cabang yang berbeda ke satu tempat.
- Memastikan bahwa kode yang ada di cabang utama selalu terupdate dengan fitur dan perbaikan terbaru.

CONTOH: git merge <nama branch yang ingin digabungkan ke 'main'>
