1. Beralih ke Cabang Utama: Pastikan Anda berada di cabang utama (biasanya main atau master). Jika tidak, gunakan perintah git checkout untuk beralih ke cabang utama.
2. Tarik Perubahan Terbaru: Pastikan cabang utama Anda terbaru dengan menjalankan git pull origin main.
3. Lakukan Merge: Sekarang, Anda dapat melakukan merge dari cabang fitur ke cabang utama dengan perintah git merge. Misalnya, jika Anda ingin menggabungkan cabang fitur-baru ke cabang utama (main).
4. Selesaikan Konflik (jika ada): Jika ada konflik dalam penggabungan, Anda perlu menyelesaikannya secara manual. Git akan menunjukkan file-file yang menyebabkan konflik, dan Anda harus memperbaikinya.
5. Commit Merge: Setelah semua konflik diselesaikan, lakukan commit untuk menyelesaikan proses merge.
6. Push Perubahan: Terakhir, push perubahan ke repositori jarak-jauh (remote repository) agar perubahan tersebut tersedia untuk kolaborator lainnya.
